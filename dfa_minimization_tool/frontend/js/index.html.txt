      <div class="theory-grid">
        <div class="theory-card theory-card-blue">
          <div class="theory-card-icon">🔍</div>
          <h3>What is DFA Minimization?</h3>
          <p>DFA minimization transforms a Deterministic Finite Automaton into an equivalent DFA with the <strong>fewest possible states</strong>. The result accepts exactly the same language — just more compactly.</p>
          <p>Every regular language has a unique (up to isomorphism) minimal DFA called the <em>canonical DFA</em> for that language.</p>
        </div>
        <div class="theory-card theory-card-green">
          <div class="theory-card-icon">⚡</div>
          <h3>Why Does It Matter?</h3>
          <ul>
            <li><strong>Efficiency:</strong> Fewer states = faster execution and less memory.</li>
            <li><strong>Uniqueness:</strong> The minimal DFA is canonical — useful for language equivalence checking.</li>
            <li><strong>Optimization:</strong> Compilers minimize DFAs for faster lexical analysis.</li>
            <li><strong>Clarity:</strong> Simpler automata are easier to reason about.</li>
          </ul>
        </div>
        <div class="theory-card theory-card-purple">
          <div class="theory-card-icon">📋</div>
          <h3>The Table-Filling Algorithm</h3>
          <p>Also called the <em>Mark Algorithm</em> or <em>Myhill-Nerode method</em>:</p>
          <ol>
            <li><strong>Remove Unreachable States</strong> — delete states never reachable from q₀.</li>
            <li><strong>Initial Marking</strong> — mark (p, q) where one is final, one is not.</li>
            <li><strong>Iterative Refinement</strong> — mark (p, q) if δ(p,a) and δ(q,a) are already marked.</li>
            <li><strong>Merge Equivalents</strong> — unmarked pairs form equivalence classes.</li>
            <li><strong>Build Minimal DFA</strong> — each class becomes one state.</li>
          </ol>
        </div>
        <div class="theory-card theory-card-orange">
          <div class="theory-card-icon">🧩</div>
          <h3>Key Concepts</h3>
          <dl>
            <dt>Distinguishable States</dt>
            <dd>States p and q are distinguishable if there exists a string w where exactly one of δ*(p,w), δ*(q,w) is a final state.</dd>
            <dt>Indistinguishable States</dt>
            <dd>States no string can tell apart — they can always be safely merged.</dd>
            <dt>Equivalence Class</dt>
            <dd>A set of mutually indistinguishable states collapsed into one state in the minimized DFA.</dd>
            <dt>Myhill-Nerode Theorem</dt>
            <dd>The number of states in the minimal DFA equals the number of equivalence classes of the Myhill-Nerode relation.</dd>
          </dl>
        </div>
      </div>
      <div class="theory-complexity-note">
        <span class="complexity-badge">⏱ Time Complexity</span>
        The table-filling algorithm runs in <strong>O(n² · |Σ|)</strong> time, where n = number of states and |Σ| = alphabet size — making it efficient for typical automata.
      </div>
    </div>
